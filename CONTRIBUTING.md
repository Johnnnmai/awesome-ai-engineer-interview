# Contributing

Thank you for helping make this the most comprehensive AI engineer interview resource on GitHub!

## How to Contribute

### Adding a Resource

1. Fork this repository
2. Add your resource in the appropriate section of `README.md`
3. Follow this format: `- [Resource Name](URL) - One-sentence description ending with a period.`
4. Ensure the resource is:
   - Actively maintained (updated within 12 months)
   - Genuinely useful for AI/ML interview prep
   - Not duplicating an existing entry
5. Submit a Pull Request

### Interview Debriefs

We especially welcome anonymized interview debriefs. Use this template:

```markdown
### Debrief #N: [Role Level] at [Company Type] ([Outcome])

**Timeline:** [Application to offer/rejection timeline]

**Rounds:**
1. **[Round Type] ([Duration]):** [What was covered, key questions asked]
2. ...

**What Worked:** [What you did well]

**What I'd Change:** [What you'd do differently]
```

**Rules for debriefs:**
- Anonymize company and interviewer names
- Modify specific details that could identify the company if needed
- Focus on the learning value, not complaints
- Include specific questions asked (the most valuable part)

### Translating to Chinese

Help us expand the `zh/` directory:
- Translate questions and answer frameworks
- Adapt cultural context where needed (e.g., interview norms differ between US and China)
- Keep technical terms in English where that's standard in the industry

## Quality Standards

- No affiliate or tracking links in community-curated sections
- Free resources preferred; paid resources clearly marked
- Each entry must have a concise, factual description
- Maintain alphabetical ordering within subsections where applicable

## Code of Conduct

Be respectful. This is a community resource. Constructive feedback is welcome; personal attacks are not.
