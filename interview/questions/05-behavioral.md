### Additional AI-Specific Behavioral Questions

- Describe a time when a prompt engineering change had an unexpected negative effect in production. How did you detect it, and what did you do?  
  *STAR hint: Situation — live system serving real users. Task — identify regression source. Action — diff prompt versions, add eval harness, roll back and retest. Result — regression caught, eval layer added to deployment gate.*

- Tell me about a project where you had to balance model capability against data privacy requirements. What trade-offs did you make and why?  
  *STAR hint: Situation — sensitive user data (PII, health records, financial). Task — build a useful AI feature without violating privacy policy. Action — chose smaller on-prem model, implemented PII scrubbing, or used differential privacy. Result — feature shipped compliant, measured accuracy delta vs. full-data baseline.*

- Describe a time you had to explain why an AI system made a wrong or unexpected decision to a non-technical stakeholder. How did you frame it and what did you change?  
  *STAR hint: Situation — system error visible to a customer or executive. Task — restore trust without over-promising. Action — used concrete examples, avoided jargon, proposed a specific fix with timeline. Result — stakeholder aligned, monitoring added to prevent recurrence.*

- Tell me about a time you evaluated multiple LLMs or AI tools for a production decision. What criteria did you use and what did you choose?  
  *STAR hint: Situation — selection decision with real cost/quality implications. Task — pick the right model/tool under constraints. Action — built a benchmark dataset, measured latency/cost/accuracy, involved stakeholders. Result — decision made with data, documented trade-offs for future reference.*

- Describe a situation where you pushed back on a request to add an AI feature because the risk or uncertainty was too high. How did you make that case?  
  *STAR hint: Situation — stakeholder request for AI feature (auto-response, automated decision). Task — assess readiness and communicate risk. Action — ran red-team tests, quantified failure modes, proposed a safer phased rollout. Result — scope reduced or timeline extended, avoided a potential harmful deployment.*
